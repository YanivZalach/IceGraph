# IceGraph – Developer Guide

## Architecture overview

```
icegraph/
├── frontend/           # Vite + React SPA (source)
│   ├── src/
│   │   ├── pages/      # HomePage, GraphPage
│   │   └── components/ # ControlBar, NodeInfoPanel, MetadataPanel, ErrorBanner
│   ├── vite.config.js
│   └── package.json
├── static/
│   └── react/          # Built frontend output (git-ignored, generated by npm run build)
├── icegraph/
│   ├── main.py         # Flask server
│   ├── icegraph_visualizer.py
│   └── js_inject.html  # Legacy UI injected into vis-network graph pages
├── Dockerfile
└── how_to_use.md
```

The application has two serving modes:

| Mode | React | Flask | Who serves the frontend |
|------|-------|-------|------------------------|
| **Development** | `npm run dev` on :5173 | `python main.py` on :5000 | Vite dev server (HMR, fast refresh) |
| **Production / Docker** | `npm run build` (once) | Flask on :5000 | Flask (`static/react/`) |

---

## Development mode

Run both servers simultaneously — each in its own terminal.

### Terminal 1 — Flask backend

```bash
cd icegraph          # repo root
python icegraph/main.py
```

Flask listens on `http://localhost:5000`.
It handles `/generate` (the Spark graph builder) and `/lib/` (vis-network assets).

### Terminal 2 — React frontend

```bash
cd icegraph/frontend
npm install          # first time only
npm run dev
```

Vite listens on `http://localhost:5173`.

Open **`http://localhost:5173`** in your browser.

The Vite dev server proxies `/generate` and `/lib/` to `http://localhost:5000` automatically
(configured in `vite.config.js`), so the form submits to Flask and the graph renders normally —
no CORS issues, no manual port switching.

> **HMR** (Hot Module Replacement) is active: React component changes appear instantly
> in the browser without a page reload.

## Production / Docker build

The Dockerfile has three stages:

1. **Node stage** — installs npm deps and runs `npm run build` inside `frontend/`.
   Output lands in `icegraph/static/react/`.
2. **uv stage** — builds the Python virtual environment with `uv sync`.
3. **Runtime stage** — minimal Python image; copies the venv, the app code,
   and the pre-built React bundle from stages 1 and 2.

```bash
# Build the image
docker build -t icegraph .

# Run it
docker run -p 5000:5000 --env-file .env icegraph
```

Everything is served by Flask on `http://localhost:5000`.
No Node or Vite process runs inside the container.

## Rebuilding the frontend after UI changes

If you change anything under `frontend/src/` and want to test it through Flask
(instead of the Vite dev server):

```bash
cd icegraph/frontend
npm run build
```

The new bundle overwrites `icegraph/static/react/`.
Restart Flask if it is already running, then open `http://localhost:5000`.

> `static/react/` is a build artefact — do not commit it to git.
> Add it to `.gitignore` if it isn't already.

## Key routes

| URL | Served by | Description |
|-----|-----------|-------------|
| `GET /` | Flask → `static/react/index.html` | React SPA (home form) |
| `GET /graph` | React Router | Graph placeholder page |
| `POST /generate` | Flask | Runs Spark, returns graph HTML |
| `GET /lib/<path>` | Flask | vis-network assets (legacy graph pages) |
| `GET /app`, `/app/*` | Flask → `static/react/` | Alias for the React SPA |

---

## Adding a new React component or page

1. Create the file under `frontend/src/components/` or `frontend/src/pages/`.
2. Wire it into `App.jsx` (new `<Route>`) or import it into an existing page.
3. In **dev mode** the change is visible immediately via HMR.
4. Before committing, run `npm run build` and verify the page loads via Flask.

---

## Environment variables

Flask reads `.env` at startup via `python-dotenv`.
The React frontend has no runtime env vars — all configuration is baked in at build time
via Vite's `import.meta.env` mechanism (prefix with `VITE_` to expose to the browser).
